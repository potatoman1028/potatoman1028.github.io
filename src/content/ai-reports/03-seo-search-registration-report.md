# 검색 엔진 최적화(SEO) 및 검색 등록 리포트

블로그의 방문자 유입을 활성화하기 위해 구글(Google Search Console)과 네이버(Naver Search Advisor)에 사이트를 등록하고 검색 엔진 최적화 작업을 수행했습니다.

---

## 1. 개요 및 계획 (Plan)
- **현황**: 신규 블로그 생성 후 검색 엔진(구글, 네이버)에 사이트가 등록되지 않아 유입이 없는 상태.
- **수집 가이드 설치**: `robots.txt`를 통해 검색 로봇에게 사이트 구조 및 사이트맵 위치 안내.
- **소유권 인증 계획**: 각 포털에서 요구하는 파일 업로드 또는 메타 태그 삽입 방식을 통해 실소유자임을 증명.
- **빌드 환경 최적화**: Astro 프로젝트가 GitHub Pages의 기본 Jekyll 엔진과 충돌하지 않도록 설정.

---

## 2. 작업 과정 및 워크플로우 (Workflow)

### Step 1: 검색 로봇 안내판(robots.txt) 설치
- `public/robots.txt` 파일을 생성하여 모든 검색 로봇의 접근을 허용하고 `sitemap-index.xml` 위치를 명시함.

### Step 2: 구글 서치 콘솔(Google Search Console) 인증
- 사용자로부터 받은 인증용 HTML 파일명 및 내용을 바탕으로 `public/google[인증코드].html`을 생성.
- 로컬 커밋 및 푸시를 통해 실제 서버에 배포 완료.

### Step 3: 깃허브 액션(GitHub Actions) 빌드 트러블슈팅
- **문제 발생**: 배포 시 Jekyll 빌드 엔진이 Astro 소스 파일을 분석하려다 오류 발생.
- **해결 1**: `public/.nojekyll` 파일을 추가하여 깃허브의 기본 빌드 시도를 차단함.
- **해결 2**: GitHub 저장소 설정(`Settings > Pages`)에서 빌드 소스를 `GitHub Actions`로 변경하도록 안내.

### Step 4: 네이버 서치어드바이저(Naver Search Advisor) 인증
- **1차 시도 (파일 방식)**: 파일은 정상 배포되었으나 네이버 측에서 인식이 지연됨.
- **2차 시도 (태그 방식)**: 가장 안정적인 HTML 메타 태그 방식을 채택하여 `src/components/Head.astro`에 직접 삽입. (개인 식별 코드는 마스킹 처리)
- 최종적으로 네이버 크롤러가 사이트 헤더에서 인증 정보를 즉시 읽을 수 있도록 조치함.

---

## 3. 최종 결과 (Results)
- **기술적 세팅 완료**: `robots.txt`, `Sitemap`, `.nojekyll` 설정이 모두 정상 반영됨.
- **검색 소유권 확보**: 구글 및 네이버에서의 소유권 인증을 위한 모든 코드적 준비가 완료되었으며, 실제 서버(https://potatoman1028.github.io)에서 확인 가능함.
- **검색 노출 대기**: 기술적 기반이 마련됨에 따라 이제 작성되는 모든 포스팅은 검색 엔진에 의해 자동 수집 및 노출되는 상태임.

---

## 4. 결론
사이트의 검색 가시성을 위한 모든 필수 기술적 조치가 완료되었습니다. 이제 작성하시는 모든 블로그 포스팅은 검색 엔진에 의해 자동으로 수집되어 잠재적 방문자들에게 노출될 준비가 되었습니다.
